Cabeçalho e corpo

Escala da tipografia

Toda a tipografia é baseada em duas variáveis Less no arquivo preboot.less: @baseFontSize e @baseLineHeight

Estas variáveis, com alguma ajuda matemática é usado como base para criar margens, paddings e line-heights para toda tipografia

Tabelas Para dados tabulares

Markup de uma tabela

Tag Descrição
<table> Elementos exibidos de uma forma tabular
<thead> Elemento principal para cabecalhos de tabela para linhas (<tr>)
<tbody> Container para elementos das linhas da tabela (<tr>) no seu corpo
<tr> Elemento pai para células da tabela (<td> ou <th>) que aparecem em uma única linha
<td> Padrão para célula da tabela
<th> Célula da tabela especial para coluna (ou linha, dependendo do escopo) Deve estar dentro de um <thead>
<caption> Uma descrição do conteúdo que a tabela possui, especialmente útil para leitores de tela 
<table>
  <thead>
    <tr>
      <th>…</th>
      <th>…</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>…</td>
      <td>…</td>
    </tr>
  </tbody>
</table>

Formulários

HTML e CSS flexíveis

A melhor coisa sobre os forms no GloboCMS Bootstrap é que todos os inputs e controles de formulários ficam bem apresentáveis independente de como você os controlas no markup. Não é necessário nenhum HTML desnecessário

Para layouts de formulários mais complexos há classes para fácil estilização e bind de eventos, para toda a etapa de envio de informações

Quatro layouts inclusos

O Bootstrap vem com quatro tipos de layout para formulários

  • Vertical (padrão)
  • Busca
  • Inline
  • Horizontal

Diferentes tipos de form para layouts requer algum markup extra, mas os controles permanecem com o mesmo comportamento

Controle dos estados e mais

Os forms do Bootstrap incluem estilos para todos os controles básicos como input, textarea, e select. Mas além disto ele contem componentes customizados como inputs informativos e suporte para lista de checkboxes

Estados como erro, aviso, e sucesso são incluídos para cada controle de formulário. Também é incluído estilos para controles desabilitados

Quatro tipos de forms

O Bootstrap tem um markup simples para os quatro estilos de form

Nome Classe Descrição
Vertical (padrão) .form-vertical (Não obrigatório) Estagnado, alinhado a esquerda e os labels sobre os elementos
Horizontal .form-horizontal Flutuando a esquerda, alinhado a direita e labels na mesma linha dos elementos do formulário
Inline .form-inline Alinhado a esquerda com um label e controles dispostos de uma forma inline-block para um estilo compacto
Busca .form-search Input de texto arredondado para uma busca típica para uma busca estética

SelectBox Personalizado

Há a possibilidade de deixar os selectboxes personalizados, usando CSS e o plugin do jQuery custom selectbox

Para deixá-lo desta forma, inclua o plugin selectbox e use o seguinte código:

$('.select-custom').selectbox();

E no markup:

<div class="control-group">
  <label class="control-label" for="multiSelect">Select personalizado</label>
  <div class="controls">
    <select id="selectCustom" class="select-custom">
      <option>personalizado 1</option>
      <option>personalizado 2</option>
      <option>personalizado 3</option>
      <option>personalizado 4</option>
      <option>personalizado 5</option>
    </select>
  </div>
</div>
Checkbox Personalizado

Há a possibilidade também de deixar os checkboxes personalizados, usando CSS e o plugin do jQuery custom input

Para deixá-lo desta forma, inclua o plugin customInput e use o seguinte código:

$('.custom-checkbox').customInput();

E no markup:

<div class="control-group">
  <div class="controls">
    <div class="checkbox">
      <input type="checkbox" id="optionsCustomCheckbox" value="option1" class="custom-checkbox">
      <label for="optionsCustomCheckbox">Opção um de um checkbox personalizado</label>
    </div>
  </div>
</div>

O Bootstrap requer markup extra em volta dos checkboxes e radio buttons para deixá-los estagnados. Agora, é uma maneira simples de repetir <label class="checkbox"> que circunda os <input type="checkbox">

Botões e ações

Como convenção, botões devem ser usado par ações enquanto hyperlinks devem ser usados para objetos. Por exemplo, um botão de Download deveria ser um botão enquanto "Atividades recentes poderia ser um link"

Âncoras e formulários

Estilos de botões podem ser aplicados em qualquer coisa com o .btn. Entretanto, tipicamente você irá querer aplicar somente em elementos <a> e <button>

Nota: Todos botões devem incluir a classe .btn. Estilos de botões podem ser aplicados em elementos <button> e <a> para consistência

Construído com sprite

Ao invés de fazer um request para cada imagem, usamos um sprite onde há todas as imagens e assim usamos background-position

Glyphicons é uma biblioteca open source de ícones

Como usar

No Bootstrap 2.0, optamos por usar uma tag <i> para todos ícones com a classe base .icon

<i class="icon search"></i>

Casos de uso
